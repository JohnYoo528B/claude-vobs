# ✍️ 日志与笔记

> **做→记**。不是思考后的产出，而是做事过程中的记录。

## 跟 1主题研究 的区别

| | 2日志与笔记 | 1主题研究 |
|--|-----------|----------|
| 驱动方式 | 做→记 | 想→写 |
| 内容性质 | 过程记录、原始数据 | 分析判断、方法论 |
| 典型例子 | 今天写了什么代码、买了哪只股票 | 这个策略为什么有效、市场规律的框架 |

**判断标准**：如果你当时没做这件事，就不会写这条笔记 → 放这里。

## 子文件夹建议

| 文件夹 | 内容 | 记录什么 |
|--------|------|---------|
| `VibeCoding开发日志/` | AI 辅助开发的项目记录 | 每个项目一个文件夹，记录每天的开发进展和决策 |
| `交易/` | 交易记录 | 买卖记录、持仓变化、当日市场观察 |
| `学习/` | 备考或学习过程 | 每日练习量、错题笔记、课程笔记 |

你可以根据自己的需求增减子文件夹。

## 开发日志

开发日志由 AI 自动管理。说「更新日志 + 项目名」触发。

## 跟主题研究的边界示例

以交易为例：
- **交易日志** → 这里（`交易/`）：每天做了什么操作，做→记
- **策略复盘** → `1主题研究/金融/`：提炼规律、系统性反思，想→写
