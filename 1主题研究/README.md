# 📖 主题研究

> **深读产出层**。这里放的不是别人的东西，是你读完之后自己的判断、方法论和框架。

## 跟草稿箱的关系

```
外部资料 → ～草稿箱/（unread）→ 读完消化 → 1主题研究/（自己的笔记）
```

草稿箱里的 = 别人的原文、原始链接。这里的 = 你的分析、你的判断、你提炼的方法论。

## 什么应该放这里

| ✅ 放这里 | ❌ 不放这里 |
|-----------|------------|
| 读完一篇论文后自己的笔记 | 论文原文 PDF 或链接（→ 草稿箱） |
| 一个投资策略的系统性复盘 | 每天的买卖记录（→ 2日志与笔记/） |
| 编程方法论、架构思考 | 开发过程的日记（→ 2日志与笔记/） |
| 某个领域的深度分析 | 临时想法（→ 草稿箱） |

**判断标准**：这是「想→写」的产物，还是「做→记」的产物？前者放这里。

## 怎么浏览

**不要点文件夹。点 MOC（主题索引）：**

MOC 是 Map of Content 的缩写——一个 Markdown 文件，用 `[[双向链接]]` 列出该主题下所有相关笔记。一篇笔记可以同时出现在多个 MOC 里。

### 创建你的第一个 MOC

1. 在 `1主题研究/` 下新建一个 .md 文件（如 `金融.md`）
2. 用 `templates/MOC模板.md` 的格式
3. 每次在这个主题下写新笔记，手动或让 AI 帮你加到 MOC 链接列表里

## 子文件夹

子文件夹是**物理存放位置**，不是分类标准。笔记放哪个文件夹不那么重要——入口是 MOC，不是文件夹。

建议按领域建子文件夹（如 `金融/`、`编程/`、`设计/`），但不用太纠结——AI 搜索比文件夹分类快。

## 新建笔记时

1. 写标题和内容，打上标签
2. 物理位置 AI 帮你放
3. AI 自动判断是否需要加入现有 MOC，拿不准时问你

## 新建 MOC

当一个主题方向的笔记积累到 3 篇以上，且难以归入现有 MOC 时，新建一个 MOC 文件（和同名物理文件夹）。
