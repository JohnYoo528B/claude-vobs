## 九、使用方式

### 对于 AI

1. 改动完成后，查本表：自己的改动类型命中哪些行？
2. 按「执行顺序」列逐行执行——数字小的先做
3. 每完成一行，用「检查方法」列验证
4. 如有行因外部条件不适用（如 VSCode 仓库不可达），标注原因并写入哨兵

### 对于 review-creator

Phase 7（收尾）自动读取本表，逐行检查命中项是否已执行。未执行 → 补执行。

### 对于操作收尾清单

各触发末尾追加 `→ 详见 [[下游传播表]]`，本文档是「怎么做」的细节。

## 十、维护规则

- **新增下游影响**：在对应分类区追加新行，不要插入到不相关的分类
- **修改现有规则**：直接修改对应行的「更新内容」列
- **废弃规则**：整行标注 `~~删除线~~` + 末尾加 `（vX.X 废弃）`，保留一个月后清理
- **版本追踪**：修改本文档后在修改记录中追加一行
