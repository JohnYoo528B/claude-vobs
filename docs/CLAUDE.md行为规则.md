# CLAUDE.md 行为规则

## 三个功能

### 1. 对话→笔记存档
- 触发词：`存成笔记` / `存档` / `保存对话`
- 流程：提炼 3-5 条 → 判断文件夹 → 生成预览 → 用户确认 → 写入

### 2. 会话连续性
> 详见 [[会话连续性规范]]。

### 3. 智能链接建议
- 自动触发：写完笔记后扫描 vault
- 高匹配（核心概念 ≥2）自动加，低匹配询问
- 格式：`---` 分隔线 + `> 📎 相关笔记：[[A]] · [[B]]`

### 4. Skills 强制预读
> 借鉴 Fable 5 "unconditional check" 机制。创建文件、写入内容、运行 bash 之前，必须先读对应 SKILL.md。
> （参考：Claude Fable 5 System Prompt 分析，原分析笔记未纳入公开版）

- **无条件触发**：任何文件创建/写入/bash 操作前，先确认是否需要读 SKILL.md
- **判断标准**：操作涉及 skill 覆盖的领域（归档/回顾/日志/搜索/清理）→ 必须先读
- **禁止跳过**：不能因为"上次读过"而跳过，每次操作前独立判断
- **边界**：纯读操作（grep/Read/搜索）不触发，只覆盖写操作

## 设计决策
1. **纯 Markdown 优于 Dataview**：零插件依赖，100% 可靠
2. **触发词匹配自然语言**：选用户自然会说的词
3. **按会话分文件防冲突**：每次会话 Write 新建独立文件，零并发写入风险
4. **双写分工明确**：快速接力 vs 永久留存，各司其职
   - `.claude/上次会话.md` → Write 覆盖（只保留最近一次）
   - `模型对话记忆/<YYYY-MM-DD-HHMM-主题>.md` → Write 新建（每次会话独立文件）

## 自动修复授权（三层响应体系）

> v4.7 起从二层升级为三层，新增"主动询问"中间层——借鉴 Fable 5 "无条件搜索不先问用户"的思路，在检测和等待触发词之间插入主动询问。

### 第一层：自动修复 + 通知（无需额外授权）
- project 字段与文件夹名不一致 → 以文件夹名为准修正
- frontmatter YAML 格式错误 → 修正缩进/引号配对
- 同文件内路径引用断裂 → 修正为目标文件存在路径

### 第二层：主动询问（检测到问题后主动问要不要修）
> 不等用户说触发词，AI 发现问题后直接问。降低认知负担，但不越过安全边界。

- Wiki 断链 → "发现 3 个断链，要不要现在修？"
- 草稿积压（≥5 个 unread） → "草稿箱有 5 篇没读，要不要清理？"
- 日志沉寂（>7 天无更新） → "xx 项目 7 天没日志了，记一下进展？"
- 跨文件路径不匹配 → "两端路径不一致，要不要对齐？"

### 第三层：提醒 + 等待触发词（当前模式）
- 记忆冗余检测（system-watch 已覆盖的周期性检查）
- 需要人工判断的架构级问题

## AI 产出质量标准

> 引用 Fable 5 版权规则作为质量检查基准。（参考：Claude Fable 5 System Prompt 分析，原分析笔记未纳入公开版）

### 引用规则
- **单条引用 ≤ 15 词**（15+ 视为违规）
- **每源最多引用 1 次**
- **歌词/诗歌完全禁止**（包括一行）

### 适用场景
- AI 生成的新笔记内容
- AI 整理/归档后的结构化输出
- AI 从外部资源消化后的产出
