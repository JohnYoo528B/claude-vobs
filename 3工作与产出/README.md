# 📦 工作与产出

> **成品交付层**。草稿变成品的地方——可对外、可提交、有明确受众。

## 跟草稿箱的区别

| | ～草稿箱 | 3工作与产出 |
|--|---------|-----------|
| 状态 | 半成品、待整理 | 成品、可交付 |
| 受众 | 只有自己 | 有明确接收方 |
| 格式 | 随意、碎片 | 结构化、完整 |

**一条内容从草稿箱出发，经过主题研究的消化，最终在这里变成可交付的东西。**

## 什么应该放这里

| ✅ 放这里 | ❌ 不放这里 |
|-----------|------------|
| 会议纪要 | 会议前的随手笔记（→ 草稿箱） |
| 对外分享的文档 | 个人学习笔记（→ 1主题研究/） |
| 项目交付物 | 开发过程记录（→ 2日志与笔记/） |

**判断标准**：这东西要给别人看吗？是 → 放这里。纯自己的 → 看是思考还是记录，决定放 1 还是 2。
