## AI 行为准则

### 这是笔记库，不是代码工作区
- **首要任务**：整理、搜索、关联、总结笔记内容
- **不要主动写代码**：除非用户明确要求在笔记里记录代码片段或开发思路
- **Markdown 优先**：所有输出用 Markdown 格式，方便 Obsidian 渲染和链接
- **善用 Obsidian 特性**：双向链接 `[[]]`、标签、frontmatter、Callout 等
- **操作后收尾**：完成任何写操作后，对照 [[_系统管理/设计规范/核心设计规范/操作收尾清单]] 检查是否触发收尾条件。命中即执行，不跳过。

### 沟通规则
- 消息中出现「明白吗 / 明不明白 / 是否明白」时：**视为最高优先中断。立即停止所有动作，先用自己的话解释理解了什么，等用户确认「对」后再执行。任何情况下不得跳过。**
- 消息中出现「存一下 / 暂存 / 存一个」时：**视为硬性 quick-capture 指令。不加工、不精修、不整理——原文照存到 `～草稿箱/手机暂存/`。即使消息中同时含有"精修/修改/整理"等动作词，也以"存一下"为准，只存不做。**
- **写时间戳前必须跑 `date` 命令**：会话记录、历史记录、文件 frontmatter 中的时间，一律以 `bash: date` 输出为准，不凭感觉写
- **改动后自检**：涉及系统架构、skill 变更、名称/文件夹变动时，改动完成后自动执行四项检查：
  1. **路径断裂**：通读所有受影响的文件，检查路径引用是否断裂、逻辑是否矛盾、命名是否一致，有则当场修复
  2. **Wiki 链接解析**：以 `[[_系统管理/系统健康/Wiki链接健康报告]]` 为基线，对修改/新建的文件，Grep `[[...]]` 提取所有 Wiki 链接，逐个校验目标文件存在性，只报告基线后新增的断链。断链当场修复或标记。
  3. **Frontmatter 校验**：对修改/新建的 .md 文件，校验 YAML frontmatter 格式（缩进、引号配对、必填字段），不合法当场修复
  4. **原理更新判断**：自问「这次改动是否涉及 `系统设计原理.md` 已有描述的变化？或新增了原理文件中未记录的系统性机制？」。是 → 告知用户「⚠️ 这次改动涉及系统设计原理的变更，这意味着需要走完整版本更新流程：更新原理 → 生成版本历史记录 → 更新使用指南 → 记忆一下。要继续吗？」。否 → 什么都不说。

### 工作流程
1. **搞清楚需求**：用户想整理什么、总结什么、查找什么
2. **并发搜索两处**：vault 笔记（Grep/Glob）+ 对话记忆（JSONL），两处结果同时呈现，让用户选要看哪个
   - 用户说"找/搜索/查一下/有没有关于 xxx"→ 必须同时搜 vault 和 `~/.claude/projects/E--2-Notes-0GithubLibraryForObsidian-John-sDataForObsidian/*.jsonl`

3. **执行**：整理、提炼、关联
4. **解释结果**：做了什么、为什么这样做
5. **外部资源自动摄入**：用户丢过来链接/文件时，自动判断类型（GitHub/PDF/文章），存入 `～草稿箱/`。必须在 frontmatter 写入 `source`（原始链接）+ `prerequisites`（读这篇之前需要先懂的概念清单）。仅在归属真拿不准时才问用户。

	> **消化后升级到 1主题研究/ 时**：AI 先判断内容本质（技术实现/概念分析/方法论/市场判断），再给出 2-3 个可能的 MOC 归属（如「建议放科技与AI/，但它的方法论也适合金融 MOC」），用户做选择。笔记写入后自动更新对应 MOC 文件的索引。物理文件夹只是存放位置，入口是 MOC。详细规则见 [[1主题研究/README]]。

> 操作步骤见各 skill 的 SKILL.md（`~/.claude/skills/<skill名>/SKILL.md`），此处仅做路由。
> ⚠️ **手机端记忆拦截**：手机端（cc-connect/微信）收到 `记忆一下` / `保存记忆` 等记忆请求时，**不直接执行**，先回复确认句（如「确定要把这段对话记下来？会存到上次会话和月记忆里」），等用户确认后再写。
> 💡 **手机端备忘**：

## 会话连续性

### 启动时
读完 CLAUDE.md 后：

**日期检测**（先执行，有提醒则暂停后续步骤）：
1. 运行 `date "+%u %d"` 获取星期几（1=周一）和日期
2. **回顾提醒合并逻辑**（两者满足其一即提醒）：
   - 今天是周一（%u=1）**且** 距上次回顾 >7 天 → "📅 今天周一，距上次回顾已 N 天，要不要更新历史记录？"
   - 距上次回顾 >14 天（任意天）→ "🕐 距上次系统回顾已 N 天，要不要更新历史记录？"
3. **1 号**（%d=01）→ 提醒："📅 今天月初，要不要更新历史记录？说「更新历史记录」就行。"

**常规检查**：AI 启动时自动执行全部 8 项检查（哨兵/降级/待办/健康/Git/依赖/MEMORY 冲突），静默执行，仅异常时提醒。待办呈现时逐条核实状态，规则见 [[待办事项]]「规则」节。
> 详细检查步骤见 system-watch SKILL.md 五项检测 + [[跨端共享规范]] §五「路径对齐校验」+ §十「错误处理」哨兵/降级机制。任务扫描格式见 CLAUDE.md「启动时」保留模板（⬇）。

### 结束时（触发词：`记忆一下` / `把这段记忆下来` / `保存记忆` / `结束` / `收工`）

> ⚠️ **用户说「结束」/「收工」时，记忆步骤自动执行，禁止再问「要不要记忆一下」。** 收工流程里记忆不是可选项。

1. 1-3 句话总结本次会话做了什么
2. **扫描待办**：Grep `status: (todo|doing)` + 读取 [[待办事项]] → 将结果填入「未完成」列表（不要只靠回忆）
3. 列出未完成事项
4. **去重检查**：写入前对比 `.claude/上次会话.md` 和当月 `_系统管理/模型对话记忆/<YYYY-MM>.md` 已有内容，跳过已记录的重复项
5. **双写**：
   - `.claude/上次会话.md`（覆盖写，快速接力）
   - `_系统管理/模型对话记忆/<YYYY-MM>.md`（同一月内追加，永久留存，可搜索）
6. **确认回复**：写入完成后，回复一段确认文字，包含"做了什么"总结 + 两个文件的路径
7. **记录 Token**：追加一行到 `_系统管理/系统健康/token-usage.jsonl`。格式：`{"date":"<ISO时间戳>","end":"obsidian","session":"<简短描述>","tokens":<估算>,"project":"<项目名或null>"}`。tokens 用"会话轮数×经验值"估算，字段标注为估算。

`.claude/上次会话.md` 格式：
```markdown
---
date: <YYYY-MM-DD>
---

# 上次会话 · <月日>

## 做了什么
- 要点 1
- 要点 2

## 未完成
- [ ] 待办 1

## 关键上下文
下次接着做时的简要提示
```

`_系统管理/模型对话记忆/<YYYY-MM>.md` 格式：每次会话追加一节 `## <月日> · <主题> .. <时间>`，含做了什么 + 未完成。

### 边界情况
- 文件不存在：静默跳过
- 上次异常退出：说"上次没有正常结束"，不纠结
- 很短会话：写"本次会话无实质产出"，只写 `.claude/上次会话.md`，不写历史记录
- 连续多次"结束"：`.claude/上次会话.md` 覆盖写，历史记录追加到同月文件（不新建）

## 架构版本号规则

版本号只保留小数点后一位（`2.0` → `2.1` → `3.0`）。小修补内容直接合并到当前版本，不另开小版本号。

文件命名：`v<版本号>版本-<YYYYMMDDHHMM>.md`（精确到分钟，如 `v3.0版本-202606080255.md`）。

评审报告命名：`v<版本号>评审报告-<YYYYMMDDHHMM>.md`（精确到分钟，如 `v3.5评审报告-202606081654.md`）。存放于 `_系统管理/系统历史记录/<大版本系列文件夹>/`。

原理文件版本化：当核心设计思路发生实质变化时，当前 `系统设计原理.md` 归档到 `_系统管理/设计规范/核心设计规范/原理旧版/系统设计原理-v<起始版本>-v<结束版本>.md`。结束版本 = 触发归档的新版本号（如 v3.7 时代原理被 v4.0 替代 → 归档为 `v3.7-v4.0`）。再写新版，标注 `v<版本号> 起`（不写死结束版本）。review-creator 在做「更新历史记录」时自动判定是否需要出新版原理文件。

## 错误处理与恢复

### 统一规则（所有流水线适用）

**重试**：任何流水线步骤失败 → 等 2 秒 → 重试 1 次 → 仍失败 → fallback + 哨兵，当场告知用户原因和建议补救操作。

**超时**：单次操作超过 30 秒未完成 → 视为失败，走重试规则。

**哨兵格式**（写入 `~/.claude/failed-ops.jsonl`，一行一条 JSON）：
```json
{"date": "YYYY-MM-DDTHH:MM", "pipeline": "<标识>", "detail": "<断裂原因>", "recovery": "<建议恢复操作>"}
```

### 统一哨兵表

所有流水线统一处理：当场告知 + 写哨兵 + 下次启动提醒。

> ①② 为 VSCode 端流水线，见 `e:\6_CodeData\CLAUDE.md`。

| 流水线 | pipeline 标识 | 断裂场景 | 告知 + 哨兵内容 |
|--------|-------------|---------|---------------|
| ③ 历史记录 | `review` | review-creator 某步命令出错 | `"recovery": "说「更新历史记录」重试"` |
| ④ 移动端捕捉 | `quick-capture` | cc-connect 掉线，内容入本地暂存 | `"recovery": "检查草稿箱最后一条是否完整，不完整则从本地暂存中补"` |
| ⑤ 会话接力 | `session-checkpoint` | `.claude/上次会话.md` 写入失败或 vault 不可访问 | `"recovery": "确认 vault 可访问后说「记忆一下」重试"` |
| ⑥ 开发日志 | `dev-log` | dev-log-creator 写入失败 | `"recovery": "确认 project 字段正确后说「更新日志」重试"` |
| ⑦ 对话归档 | `archive` | 归档写入时 vault 不可访问或单文件权限不足 | `"recovery": "确认 vault 可访问后说「整理一下」重试"` |
| ⑧ GitHub 发布 | `github-release` | VSCode 仓库路径不可达、gh CLI 未认证、或 git push 失败 | `"recovery": "确认 VSCode 仓库可访问后说「发布 GitHub 新版本」重试"` |

### 静默降级
断裂时不直接失败，而是降到"本地缓存模式"，等条件恢复后自动迁移。

| 场景 | 降级行为 | 恢复触发 |
|------|---------|---------|
| vault 不可访问 | 内容写入 `~/.claude/fallback/`，文件名 = `<流水线标识>-<时间戳>.json` | 下次任意 vault 操作成功时提醒迁移 |
| 单文件写入失败 | 跳过该文件，其余继续，失败列表记入哨兵 | 哨兵处理时一起重试 |

降级文件格式：`{"pipeline": "degraded-<原pipeline标识>", "target": "<vault 内目标路径>", "content": "<暂存内容>"}`。

### 启动检查
每次会话启动时，AI 静默执行以下检查：

**哨兵检查**（`~/.claude/failed-ops.jsonl`）：
- 无文件或无未处理条目 → 跳过
- 有条目 → 简短提醒："⚠️ 上次有 N 个操作未完成，说「恢复未完成操作」处理。"
- 用户说"恢复未完成操作"后，按哨兵记录的 `recovery` 逐条处理，全部完成后清空文件

**降级恢复检查**（`~/.claude/fallback/`）：
- 目录为空或不存在 → 跳过
- 有文件 → 提醒："📦 上次有 N 条内容暂存在本地，说「恢复暂存内容」迁移到 vault。"
- 用户说"恢复暂存内容"后，逐文件写入 vault 目标路径，全部完成后清空 fallback 目录

## 断点续做

多步流程（如 review-creator 的十步操作）中途断裂时，不从头重跑。

### 机制
- 流程开始时检查 `~/.claude/checkpoints/<流程名>.json` 是否存在
- 存在且未过期（<2 小时）→ 从上次断点继续，跳过已完成步骤
- 不存在或已过期 → 从头开始
- 每个步骤完成后写入：`{"step": <步骤号>, "ts": "<完成时间>", "result": "<关键摘要>"}`

### 触发
- 用户说"断点续做" / "继续上次" → 扫描 `~/.claude/checkpoints/` 找最近的未完成流程
- 流程正常执行时自动使用，无需手动触发

## Git 安全网

任何涉及 vault 文件的写操作前（新建/修改/删除），AI 自动执行：

1. **操作前快照**：`git add -A && git commit -m "auto: 操作前快照 <操作描述>"`（静默，不告知用户）
2. **执行操作**
3. **操作失败时**：用户说"撤销刚才的操作" → `git reset --hard HEAD~1` 回到操作前状态

> 仅 vault 文件操作触发。纯读取、搜索、对话存档不走此流程。
