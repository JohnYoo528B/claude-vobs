# Vault 搜索

## 触发条件

用户说 `找 xxx` / `搜索 xxx` / `查一下 xxx` / `有没有关于 xxx` 时触发。

> ⚠️ 触发词不要求精确匹配。AI 识别到用户想查找信息时，自动走本流程。

---

## 核心原则：双源并发搜索

**必须同时搜索两个来源**，缺一不可：

| 来源 | 工具 | 内容 |
|:--:|------|------|
| 📁 **Vault 笔记** | Grep + Glob | vault 中所有 `.md` 文件的正文和文件名 |
| 💬 **对话记忆** | `conv-search.py` | 历史对话 JSONL 中的讨论内容 |

两个来源**必须并发执行**，不先后串行。两处结果同时呈现，让用户选要看哪个。

---

## conv-search.py 用法

**脚本位置**：`_系统管理/设计规范/_全局规范/conv-search.py`

### 三种模式

| 模式 | 命令 | 用途 |
|------|------|------|
| 搜索 | `python conv-search.py search "<关键词>"` | 在所有 JSONL 中搜索包含关键词的对话 |
| 提取 | `python conv-search.py extract <session-id>` | 提取指定会话的完整对话 |
| 列表 | `python conv-search.py list` | 列出所有可用的 JSONL 文件及其时间范围 |

### 时区说明

JSONL 中的时间戳是 **UTC**。北京时间 = UTC + 8 小时。呈现结果时自动换算。

### 降级方案

Python 不可用或 conv-search.py 缺失时：
1. 用 Grep 直接搜 `~/.claude/projects/E--2-Notes-0GithubLibraryForObsidian-John-sDataForObsidian/*.jsonl`
2. 告知用户"对话记忆搜索降级为直接 grep，结果可能不够精确"

## 执行步骤

### 1. 提取搜索关键词

从用户消息中去掉触发词（找/搜索/查一下/有没有），剩下的作为搜索关键词。多关键词用空格分隔。

### 2. 并发搜索两处

同时执行：
- **Vault 侧**：`Grep pattern="<关键词>"`（vault `.md` 文件，排除 `.obsidian/`、`.git/`、`node_modules/`）
- **对话记忆侧**：`Bash: python conv-search.py search "<关键词>"`

### 3. 分级呈现结果

按匹配数量分级展示：

| 匹配数 | 呈现格式 |
|:--:|------|
| 0 条（双源） | "没找到和「XXX」相关的内容。vault 和对话记忆都没有。" |
| 1-5 条 | 逐条列出：`[[路径/文件名]]` — 匹配摘要（1 行），分两区展示 |
| 6-15 条 | 列出文件名，末尾提示"结果较多，可以缩小关键词" |
| >15 条 | 给前 5 条最新修改的 + "还有 X 条，建议更具体的关键词" |

### 4. 结果格式模板

```
📁 **Vault 笔记**（N 条）
- [[路径/笔记名]] — 匹配行摘要
- [[路径/笔记名]] — 匹配行摘要

💬 **对话记忆**（N 条）
- YYYY-MM-DD — 对话摘要（session: xxx）
- YYYY-MM-DD — 对话摘要（session: xxx）

---
要查看哪个的详细内容？
```

每条结果格式：`- [[相对路径/文件名]] — 摘要`。路径用从 vault 根开始的相对路径，方便 Obsidian 渲染链接。

### 5. 交互提示

用户选择后：
- 选 vault 笔记 → 读取并展示
- 选对话记忆 → 用 `conv-search.py extract <session-id>` 提取完整对话

## 错误处理

| 场景 | 处理 |
|------|------|
| Python 缺失 | 降级为 Grep 直接搜 JSONL，标注"对话记忆搜索降级" |
| JSONL 目录为空 | 只展示 vault 结果，标注"对话记忆无数据" |
| Grep 超时 | 缩小搜索范围（限制到最近修改的 200 个文件）后重试 |
| 双源均无结果 | 提示"vault 和对话记忆都没有相关内容，试试换关键词？" |

---

## 三件事记住

1. **双源必须并发**：vault + 对话记忆，不串行
2. **呈现分两区**：📁 vault / 💬 对话记忆，清晰标注来源
3. **结果让用户选**：不替用户决定看哪个，列出后等选择
