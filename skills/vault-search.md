# Vault 搜索

## 触发条件

用户在手机端（微信/cc-connect）发送搜索请求。

## 执行步骤

### 1. 提取搜索关键词

从用户消息中去掉触发词，剩下的作为搜索关键词。

### 2. 搜索

用 Grep 在 vault 里搜索关键词，同时搜文件名和正文内容。优先搜 `.md` 文件。

### 3. 返回结果

根据匹配数量：

| 匹配数 | 回复格式 |
|--------|---------|
| 0 条 | "没找到和「XXX」相关的内容" |
| 1-5 条 | 逐条列出：`[[路径/文件名]]` — 匹配摘要（1 行） |
| 6-15 条 | 列出文件名，末尾提示"结果较多，可以缩小关键词" |
| >15 条 | 给前 5 条最新修改的 + "还有 X 条，建议更具体的关键词" |

每条结果格式：`- [[相对路径/文件名]] — 摘要`

> 路径用从 vault 根开始的相对路径，方便 Obsidian 渲染链接。

### 4. 不搜索的目录

跳过 `assets/`、`.obsidian/`、`.git/`、`node_modules/`。
