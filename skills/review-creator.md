# 历史记录创建器

## 适用场景

记录 vault 架构变更 + 资源快照到 `_系统管理/系统历史记录/`。触发词路由见 CLAUDE.md → "两个 skill 的分工" 表（权威来源）。代码级变更请用 dev-log-creator。

---

## 第一步：获取当前日期 + 版本号

```bash
date "+%Y %m %d %u %W"
```

读取 `_系统管理/系统历史记录/` 下各子文件夹（排除 `早期记录/`、`GitHub公开版/`），找到当前大版本系列文件夹（如 `v3系列版本/`），扫描其中的最新版本号。版本号递增规则：

| 变动 | 版本涨幅 |
|------|---------|
| 格局重组、不兼容变更 | 主版本 +1（如 2.x → 3.0） |
| 新增显著功能/内容 | 次版本 +0.1 |
| 小修小补 | 次版本 +0.1 |

---

## 第二步：跑磁盘扫描命令

```bash
# 磁盘总览
df -h /c/ /e/ /d/

# 代码区
du -sh <你的代码工作区路径>/ <你的代码工作区路径>/*/ 2>/dev/null

# AI 运行数据
du -sh ~/.claude/ ~/.claude/projects/ ~/.claude/file-history/ ~/.claude/plans/ ~/.claude/skills/ ~/.claude/telemetry/ 2>/dev/null

# 知识库（Obsidian Vault）
du -sh <你的 Obsidian vault 路径>/ <你的 Obsidian vault 路径>/.obsidian/ <你的 Obsidian vault 路径>/.git/ <你的 Obsidian vault 路径>/1学习笔记/ <你的 Obsidian vault 路径>/2工作文档/ <你的 Obsidian vault 路径>/_系统管理/系统历史记录/ <你的 Obsidian vault 路径>/0VibeCoding开发日志/ 2>/dev/null
```

---

## 第三步：统计 Token 用量

1. 读取 `~/.claude/projects/<工作区名>/token-usage.jsonl`
2. 筛选近期条目，统计会话数，汇总估算 token
3. 如果文件不存在或为空，标注"暂无数据（即日起开始追踪）"

---

## 第四步：收集项目进展

扫描代码工作区下所有项目文件夹，对每个项目记录：
- 文件夹名 + 大小
- 状态：14 天内修改过 → 活跃，否则 → 休眠，已标记完结 → 完结

---

## 第五步：检查 L2 记忆垃圾回收

1. 读取 `~/.claude/projects/<工作区名>/memory/MEMORY.md`
2. 找出所有标记 ✅（完结）的项目分组
3. 对每个完结项目，检查 `memory/` 下是否有 `scope: project` + `project: <该项目>` 的记忆文件
4. 如有，在后续"架构变更"表格中增加一行提醒

> ★ L2 记忆随项目完结可清理，防止记忆无限膨胀。

---

## 第六步：对比上一次记录

读取 `_系统管理/系统历史记录/` 下当前大版本系列文件夹中的最近一次版本文件，提取资源简报各项数值，与本次对比，填入"较上次"列。

异动阈值（超过时在表格下方加一行说明）：

| 指标 | 阈值 | 归因方向 |
|------|------|---------|
| AI 数据总大小 | ±5MB | 标注增长/缩减最多的目录 |
| 代码区总大小 | ±100KB | 新增/删除项目，或源码大幅修改 |
| 知识库总大小 | ±1MB | 标注变化最大的目录 |
| Token 消耗 | ±20% | 标注主要消耗项目的变化 |
| 磁盘已用 | 不归因 | 只看健康（低于 20% 提醒），变化不管 |

---

## 第七步：检查架构变更

1. 读取 `_系统管理/系统历史记录/` 下当前大版本系列文件夹中的最新版本文件
2. 对比 vault 实际状态（文件夹结构、命名规则、工作流程、skill、CLAUDE.md 等）
3. 找出差异，填入"架构变更"表格
4. 无变更 → 表格写"本期无架构变更"

---

## 第九步：更新追踪文件

```bash

```

## 三件事记住

1. **只有一个概念：历史记录** — 不再区分"周回顾""月回顾"，统一用"更新历史记录"触发
2. **磁盘数据当场跑命令获取，不存中间文件** — 每次数据是最新的
3. **Token 是估算不是精确值** — 偏差不可怕，趋势才重要
