# 对话→笔记存档

## 触发条件

用户说 `整理成笔记` / `整理一下` / `归档` / `归整` 时触发。

> ⚠️ 与 quick-capture 的边界：核心词含"整理/归档/归整/成笔记" → 走本流程（提炼结构化）。核心词含"存/暂存/记"且不含上述词 → 走 quick-capture（原文照存）。

---

## 执行流程

### 1. 回顾对话
梳理核心问题、决定、洞察。

### 2. 提炼 3-5 条结构化要点
不逐字复述，提炼核心。

### 3. 判断归属文件夹
从 `1主题研究/`、`2日志与笔记/`、`3工作与产出/` 中选择。拿不准时列出理由让用户选。

### 4. 去重检查
用 Grep 搜索 vault 中标题相似（≥70% 匹配）的已有笔记，有则提示"已有 [[xxx]]，要合并还是新建？"

### 5. 按模板生成预览
Frontmatter（`date` + `type` + `status` + `tags`）+ 正文 + `## 相关链接`。

### 6. 搜索关联笔记
用 Grep 搜索 vault 找相关笔记，填到相关链接节。

### 7. 展示预览，等用户确认后再写入

---

## 边界情况

| 情况 | 处理 |
|------|------|
| 对话超 20 轮 | 只提炼最核心 3-5 点，标注"精简版" |
| 无实质内容 | 回复"这次对话主要是操作性的，没有需要存档的讨论" |
| 话题跨领域 | 列出选项让用户选 |
| 已有类似笔记 | 提示"已有 [[xxx]]，要合并还是新建？" |

---

## 错误处理

vault 不可访问或单文件权限不足 → 写哨兵 `pipeline: "archive"`，告知用户。恢复指令：`"确认 vault 可访问后说「整理一下」重试"`。

---

## 三件事记住

1. **先预览再写入**：永远不等用户确认就写入，展示 frontmatter+正文预览后等回复
2. **去重优先**：Grep 相似标题是必须步骤，不跳过
3. **位置判断用排除法**：先看是不是工作产出（→ 3），再看是不是学习日志（→ 2），都不是 → 1主题研究
