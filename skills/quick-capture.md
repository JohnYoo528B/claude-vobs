# 快速捕捉到草稿箱

## 触发条件

用户在手机端（微信/cc-connect）发送要暂存的内容。

触发词：

| 触发词 | 行为 |
|--------|------|
| `存 xxx` / `暂存 xxx` / `记录 xxx` / `存一下 xxx` | 捕捉**当前这一条** |
| `以上都存一下` / `以上都存下来` / `以上全存下来` | 捕捉**上一次记录之后所有未记的** |

> 触发词不要求精确匹配。用户说"帮我记一下×××""把这个存下来""写进草稿箱"等，AI 应识别意图为快速捕捉，执行同样流程。注意与"笔记归档"区别——含"整理/归档/归整/成笔记"时不走本流程。

---

## ⛔ 铁律（违反即失败）

1. **原文照存**：用户发什么就存什么，禁止提炼、总结、改写、结构化。哪怕内容看起来像"需要整理的知识"，也不准加工。
2. **按对话会话分文件**：同一 Claude Code session 内所有暂存写入同一个文件；`/new` 后自动新建文件。
3. **文件名 = 时间戳 + 概要**：`<YYYYMMDD-HHMMSS>-<内容概要>.md`，时间戳取该文件**最后一条消息**的时间。

---

## 执行步骤

### 1. 获取时间戳

用 `date "+%Y-%m-%d %H:%M:%S"` 获取当前时间（精确到秒）。

### 2. 判断所属文件

- 看当前 Claude Code session ID
- `～草稿箱/手机暂存/` 下是否有同一 session 已有的文件？
  - **有** → 追加到该文件
  - **无** → 新建文件，命名 `～草稿箱/手机暂存/<最后时间戳>-<内容概要>.md`
- 内容概要从对话主题中提炼，≤15 字，文件名不含空格

### 3. 提取内容

**单条捕捉**：用户消息去掉触发词后剩下的就是正文。例如：
- "记一下 明天要买牛奶" → 正文 = "明天要买牛奶"

**批量捕捉**：当前 session 中上次记录之后所有未记过的 Q&A，按时间顺序。

如果正文为空，回复"要记什么内容？"让用户补充。

### 4. 写入文件

**路径**：`～草稿箱/手机暂存/<YYYYMMDD-HHMMSS>-<内容概要>.md`

**文件格式**：
```markdown
---
date: <YYYY-MM-DD>
tags:
  - 类型/草稿
---

# <内容概要>

- YYYY-MM-DD 简短主题：用户提问原文 + AI 回答原文
- YYYY-MM-DD 简短主题：…
```

- 按时间顺序排列，新条目追加在末尾
- 每条包含完整 Q&A，不缩略
- 主题控制在 10 字以内
- 可折行

### 5. 回复确认

```
✅ 已存到草稿箱 `～草稿箱/手机暂存/<文件名>.md`
```

一句话确认即可，不要展开文件内容。

## 错误处理

### vault 不可访问（cc-connect 掉线 / vault 路径不可达）

不直接报错丢弃。降级到本地缓存：

1. 将捕捉内容写入 `~/.claude/fallback/quick-capture-<时间戳>.json`
2. 格式：`{"pipeline": "degraded-quick-capture", "target": "～草稿箱/手机暂存/<文件名>.md", "content": "<待存内容>"}`
3. 回复用户："⚡ vault 暂时不可访问，内容已暂存本地。下次 vault 正常时会自动同步。"

> 恢复由 Obsidian 端或 VSCode 端启动时检查 `~/.claude/fallback/` 触发。

## 外部资源摄入

用户丢过来链接或文件时，AI 自动判断类型并存入草稿箱。

### 类型判定

| 类型 | 识别方式 | 示例 |
|------|---------|------|
| GitHub | URL 含 `github.com` | 仓库链接、Issue、PR |
| PDF | URL 以 `.pdf` 结尾或附件为 PDF | 论文、报告 |
| 文章 | URL 含 `medium.com`、`zhuanlan.zhihu.com` 等 | 博客、专栏 |
| 通用 | 无法归入以上类别 | 其他网页或文件 |

### Frontmatter 字段

存入 `～草稿箱/` 时，必须在 frontmatter 写入：

| 字段 | 必填 | 说明 | 示例 |
|------|:--:|------|------|
| `source` | ✅ | 原始链接 | `https://arxiv.org/abs/2106.09685` |
| `prerequisites` | 选填 | 读之前需要先懂的概念 | `做市商机制, 订单簿` |

### 存入规则

- 存入 `～草稿箱/`，标记 `status: unread`
- 归属拿不准时才问用户（明显是金融的 → 草稿箱 + 建议标签，不反复确认）
- 启动时 AI 会按前置知识难度分级提醒
